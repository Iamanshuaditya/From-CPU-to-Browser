# Contributing Guide

Thanks for contributing. This repo values practical, tested, measurable improvements.

## What to Contribute
- fixes to technical mistakes
- better experiments and benchmarks
- clearer debugging playbooks
- project milestone/test improvements
- challenge solutions with evidence

## Contribution Workflow
1. Open or pick an issue.
2. Create a focused branch.
3. Make changes with tests and measurements.
4. Run local quality checks.
5. Open PR using template.

## Pull Request Quality Bar
Every PR should include:
- problem statement
- scope of change
- before/after behavior
- test evidence
- performance evidence when relevant

## Commit Guidelines
Use clear, scoped commits:
- `docs: improve hydration case study measurements`
- `project: add keep-alive parser regression tests`
- `debugging: add triage tree for TLS issues`

## Review Expectations
Reviewers check for:
- technical correctness
- reproducibility
- clarity
- link integrity
- no hidden scope creep

## Local Validation Checklist
- [ ] Markdown renders correctly.
- [ ] Internal links resolve.
- [ ] New content has measurable acceptance criteria.
- [ ] Experiments include expected outcome and why it matters.

## Good First Contributions
- Improve one checklist with clearer measurable criteria.
- Add one reproducible performance experiment.
- Add one debugging decision tree based on real traces.
