# Project Rubric

## Correctness
- Parser/engine behavior matches expected outputs.
- Error handling works for malformed input.

## Performance
- Benchmarks include CPU, memory, and frame time data.
- Improvements are justified with before/after metrics.

## Reliability
- Stress tests cover heavy updates and low-resource scenarios.
- Regressions are tracked and fixed.

## Engineering Quality
- Clear architecture boundaries.
- Reproducible builds and scripts.
- Useful instrumentation and logs.

## Documentation
- Tradeoffs are explained.
- Results are reproducible.
- Limitations and future work are clearly stated.
